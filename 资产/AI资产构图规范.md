# 铜声·识洛 — AI 资产构图规范

> **核心原则：先画整幅，再切碎片。凡是需要在画面上拼合的图层，必须从同一张源图中裁切。**
>
> AI 生图有随机性。如果两张拼图分别生成，它们的接缝边缘、透视关系、色彩温度永远对不上。
> 解决方案：**Composition-First Generation（构图优先生成）**

---

## 一、画布坐标系

所有资产在 1920×1080 的逻辑画布上定位。谜题元素的"目标位置"是引擎中硬编码的像素坐标。

```
(0,0)──────────────────────────────(1920,0)
  │                                    │
  │         1920 × 1080                │
  │        逻辑画布 (16:9)             │
  │                                    │
  │    ┌─────────┐                     │
  │    │ 目标区域 │  ← targetRect      │
  │    │ (x,y,w,h)│                    │
  │    └─────────┘                     │
  │                                    │
(0,1080)───────────────────────────(1920,1080)
```

**关键约定：**
- 谜题元素的"对齐成功"= 元素中心进入 targetRect 中心 ±30px
- 所有嵌套/拼接关系的元素共享同一个坐标系
- 尺寸单位统一为逻辑像素（引擎自动处理 DPR 缩放）

## 二、三类资产及其生成策略

| 类型 | 定义 | 生成策略 | 数量 |
|------|------|----------|:----:|
| **A 型 · 独立帧** | 单独显示，不与任何其他帧拼合 | 直接按目标尺寸生成 | ~160 张 |
| **B 型 · 配对件** | 两个或多个元素必须在画面上对齐/拼合 | **构图优先生成** → 裁切 | ~48 张 |
| **C 型 · 嵌套层** | 多层叠加（裂缝/透视），共享同一个构图 | **母图生成** → 逐层分离 | ~12 张 |

### A 型 · 独立帧（直接生成）

这些资产单独展示，不需要与其他资产拼合。直接按目标尺寸用 Seedream 生成。

**示例：** 场景背景图、VFX 关键帧、UI 元素、人物侧写图。

**提示词末尾固定添加：**
```
画面尺寸 1920×1080 像素，16:9 画幅比。
```

### B 型 · 配对件（构图优先生成）

两个或多个元素必须在画面上对齐。必须先生成"母图"，再裁切出碎片。

### C 型 · 嵌套层（母图 + 逐层分离）

多层叠加共享同一构图。先生成最底层的完整画面，上层元素从同一构图中逐层提取。

### 教学关 L1 · 接触（门板嵌入门框）

**嵌套关系：** 门板 L1-02 (800×600) → 嵌入门框 L1-01 中的门洞

**母图策略：** 生成一张 1920×1080 的"门框完整画面"，门洞区域为透明/空白。门板从另一张 1920×1080 "门关闭完成"画面中裁切出门板区域。

```
母图 1920×1080:
┌──────────────────────────────────────┐
│                                      │
│         ┌──────────┐                 │
│         │          │                 │
│         │  门洞    │  ← 800×600      │
│         │  (透明)  │    中心点       │
│         │          │    (960,540)    │
│         └──────────┘                 │
│                                      │
└──────────────────────────────────────┘
```

| 资产 ID | 名称 | 生成方式 | 裁切坐标 | 最终尺寸 |
|:-------:|------|----------|----------|:--------:|
| L1-01 | 空门框画面 | 整张生成，门洞区域留空 | 不裁切 | 1920×1080 |
| L1-02 | 门板 | 从 L1-03 "门关闭"画面裁切 | crop(560,240,1360,840) | 800×600 |
| L1-03 | 门关闭完成 | 整张生成（含门板在门洞内） | 不裁切 | 1920×1080 |

**门洞定位约束：**
- 门洞中心：(960, 540) 即画布正中
- 门洞尺寸：800×600
- 门板尺寸：必须与门洞完全一致 (800×600)
- 门框内边距：四周各 10px 装饰边框

**生成步骤：**
1. 用 Seedream 生成 L1-03（门关闭的完整画面，门板严丝合缝在门洞中）
2. 从 L1-03 中裁切出门板区域 (560,240)→(1360,840) 得到 L1-02
3. 用 Seedream 单独生成 L1-01（同样的门框，但门洞是空的——可后期 PS 擦除门板）

### 教学关 L2 · 透视（圆形洞口嵌套）

**嵌套关系：** 前景层圆形洞口 (r=60) → 透过洞口看到后景层

**母图策略：** 前景层和后景层各自独立生成，但共享同一个"视觉焦点"坐标。

| 资产 ID | 名称 | 尺寸 | 洞口/焦点位置 |
|:-------:|------|:----:|:-------------:|
| L2-01 | 前景层（带圆形洞口的画面） | 1920×1080 | 洞口中心 (960,540), r=60px |
| L2-02 | 后景层（透过洞口看到的画面） | 1920×1080 | 视觉焦点 (960,540) |

**对齐约束：**
- 前景层洞口中心 = 后景层焦点 = 画布中心 (960,540)
- 后景层在洞口区域内的画面必须是一个完整、自洽的小场景
- 洞口半径 60px = 120px 直径（在引擎中硬编码）

### 教学关 L3 · 合力（齿轮啮合）

**嵌套关系：** 固定齿轮 + 可动齿轮啮合转动

**母图策略：** 生成一张"两齿轮啮合完成"画面，从中裁切出两个齿轮。

| 资产 ID | 名称 | 生成方式 | 最终尺寸 |
|:-------:|------|----------|:--------:|
| L3-01 | 固定齿轮+可动齿轮 | 母图生成 → 裁切 | 1920×1080 |
| L3-02 | 两齿轮啮合转动 | 直接生成（完成态画面） | 1920×1080 |

**齿轮约束：**
- 固定齿轮中心：(760, 540)
- 可动齿轮中心：(1160, 540)
- 齿轮半径：100px
- 齿数：8 齿（45° 间距）
- 两齿轮中心距：400px = 2 × 半径（刚好啮合）

### 壹·二里头 谜题 #1 · 陶范合拢

**嵌套关系：** 上半范 (1000×800) → 与下半范 (1000×800) 龙纹咬合

**⚠️ 这是全游戏第一个正式配对件，龙纹必须在接缝处精确延续。**

```
母图 1920×1080（俯视工作台+合拢的陶范）:
┌──────────────────────────────────────┐
│                                      │
│  ┌──────────────────────────┐        │
│  │         上半范           │        │
│  │   龙纹上半部分            │        │
│  │   ───接缝线───           │ ← y=540│
│  │   龙纹下半部分            │        │
│  │         下半范           │        │
│  └──────────────────────────┘        │
│         ↑ x=460          ↑ x=1460   │
│                                      │
└──────────────────────────────────────┘
```

| 资产 ID | 名称 | 生成方式 | 裁切坐标 | 最终尺寸 |
|:-------:|------|----------|----------|:--------:|
| 01-01 | 工作台+两半陶范（合拢状态） | 整张生成 | 不裁切 | 1920×1080 |
| 01-02 | 上半范格 | 从母图裁切上半部 | crop(460,140,1460,540) | 1000×400 → 缩放至 1000×800 |
| 01-03 | 下半范格 | 从母图裁切下半部 | crop(460,540,1460,940) | 1000×400 → 缩放至 1000×800 |

**接缝约束：**
- 接缝线：y = 540（画布正中水平线）
- 龙纹在接缝处必须连续——龙身跨越上下两半
- 龙尾在右边缘处断开（后续龙尾化绳序列的起始点）
- 上半范裁切后需纵向拉伸 2×（400→800），下半范同理
- **替代方案：** 如果拉伸变形不可接受，直接生成两张 1000×800 的范格，但必须在提示词中明确标注"接缝边缘的龙纹必须完全对齐"

**更稳妥的生成流程：**
1. 生成母图：1920×1080 俯视工作台，陶范合拢状态，龙纹完整
2. 裁切上半范 + 下半范
3. 如果需要独立的高质量范格：以上半范/下半范裁切图为参考，用 Seedream 的"图生图"功能放大到 1000×800

### 灰页 谜题 #2 · 裂缝嵌套

**嵌套关系：** 5 层裂缝画面叠加 (C 型嵌套层)

**⚠️ 这是 C 型嵌套——5 层共享同一个构图，裂缝位置必须对齐。**

```
母图 1920×1080（最深层的暗页·完整画面）:
┌──────────────────────────────────────┐
│                                      │
│    暗页完整画面                       │
│    （观星者笔记 + 张衡星图 + 秘密）    │
│                                      │
│          ┌───裂缝 A───┐              │
│         ╱              ╲             │
│        ╱   ┌─裂缝 B─┐  ╲            │
│       ╱   ╱          ╲   ╲           │
│      ╱   ╱  ┌裂缝 C┐  ╲   ╲          │
│     ╱   ╱  ╱        ╲  ╲   ╲         │
│    ╱   ╱  ╱  裂缝 D  ╲  ╲   ╲        │
│   ╱   ╱  ╱            ╲  ╲   ╲       │
│  └───┘  └──────────────┘  └───┘      │
│                                      │
└──────────────────────────────────────┘
```

**生成流程：**
1. 生成暗页母图 (1920×1080)：包含张衡星图的完整秘密画面
2. 生成 5 张覆盖层 (1920×1080)：每张是一个不同深度的灰色层，带有对应层的裂缝镂空
3. 裂缝镂空的位置必须从同一个"裂缝模板图"中裁切

| 层 | 资产 ID | 灰度 | 裂缝大小 | 备注 |
|:--:|:-------:|:----:|:--------:|------|
| 第1层 | 0G-02 | #A0A0A0 | 最小裂口 | 最上面，只看到灰色 |
| 第2层 | 0G-03 | #888888 | 稍大 | 开始隐约看到下层 |
| 第3层 | 0G-04 | #666666 | 中等 | 能看到画面局部 |
| 第4层 | 0G-05 | #444444 | 较大 | 隐约看到文字 |
| 第5层 | 0G-06 | #222222 | 最大（近乎全开） | 暗页完整显现 |

**裂缝模板约束：**
- 生成一张 1920×1080 的黑白"裂缝模板图"
- 白色 = 裂缝（镂空区域），黑色 = 保留（灰色覆盖）
- 5 个裂缝层是同一裂缝图案的 5 级"扩散"：从小到大
- 所有裂缝中心点相同：画布中心偏左 (800, 500)
- 暗金色光晕 (#8B7355) 沿裂缝边缘绘制（引擎后处理，不需要生成）

### 周·营洛 谜题 #3 · 孔老圆心对准

| 资产 ID | 名称 | 尺寸 | 圆心位置 |
|:-------:|------|:----:|:--------:|

### 东汉·纸成 谜题 #6 · 水帘对位

**嵌套关系：** 水槽格 + 帘格 + 水面格，三格 1200×800 共享尺寸

| 资产 ID | 名称 | 尺寸 | 关键区域 |
|:-------:|------|:----:|----------|
| 03c-01 | 水槽格（纸浆纤维） | 1200×800 | 纤维排列方向：水平 |
| 03c-02 | 帘格（竹帘） | 1200×800 | 帘纹方向：初始 0°（水平） |
| 03c-03 | 水面格（倒影） | 1200×800 | 倒影 = 帘格的镜像 |

**对齐约束：**
- 三格完全同尺寸 (1200×800)
- 帘格叠在水槽格上方，帘格旋转 90° 为正确
- 帘格旋转由引擎实现，不需要生成 4 个角度的图
- 水面格倒影 = 帘格垂直翻转 + 水波变形（引擎后处理）

**生成约束：**
- 水槽格：纤维必须呈明显的水平方向排列
- 帘格：帘纹（竹篾）必须呈明显的单一方向
- 水面格：平静水面 + 轻微波纹，无竹帘倒影（引擎叠加）

## 五、生成顺序规范

对于有配对关系的资产，必须按以下顺序生成：

```
1. 母图（完整场景，所有元素在正确位置）
      ↓
2. 裁切（按构图规范中的坐标裁切出各个碎片）
      ↓
3. 图生图放大（如果裁切后需要更高分辨率）
      ↓
4. 独立资产（不依赖配对的背景帧、VFX帧等）
      ↓
5. 视频（以静态帧为首帧，用 Seedance 生成动态）
```

**绝对禁止：** 先分别生成碎片，再试图让它们拼合。

## 六、尺寸标准表

### 常用尺寸规格

| 规格名 | 尺寸 | 用途 |
|--------|:----:|------|
| 全画布 | 1920×1080 | 场景背景、VFX 帧、UI 帧 |
| 宽景 | 2400×1080 | 横向滚动场景（地图/长卷） |
| 超宽 | 3600×800 | 九鼎排列等超长场景 |
| 高景 | 1200×2400 | 纵向场景（永宁寺塔） |
| 大方 | 1000×1000 | 铜镜格等正方形谜题元素 |
| 中方 | 800×800 | 孔老格等正方形谜题元素 |
| 小方 | 600×600 | 风铎格等小型方形元素 |
| 大方横 | 1200×800 | 纸成三格、人物侧写 |
| 中方横 | 1000×700 | 诗稿格、纸团格 |
| 小方横 | 800×600 | 手掌格、特写细节 |
| 扁方横 | 800×400 | 反光格、烟雾格、塔檐格 |
| 窄方横 | 600×400 | 刀背格、经架层、塔层格 |
| 小格 | 400×300 | 龙首格、塔层小格 |
| 超小方 | 400×400 | 手掌格、盖章动画、印章 |

### 尺寸选择规则

1. **独立展示的场景** → 1920×1080（全画布）
2. **需要横向滚动的场景** → 2400×1080 或 3600×800
3. **需要纵向展示的场景** → 1200×2400
4. **可拖拽的谜题元素** → 根据引擎中 targetRect 的实际像素尺寸
5. **配对件** → 按母图裁切后的目标尺寸，但母图用 1920×1080
